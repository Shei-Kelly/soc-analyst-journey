# Day 4 – Hour 3: Wazuh Alert Investigation

## Key Alert Fields

- Rule ID
- Severity Level
- Timestamp
- User
- Source IP
- Description

## Investigation Process

1. Read alert details
2. Identify affected user or system
3. Determine severity
4. Review supporting logs
5. Build timeline
6. Assess risk
7. Escalate if necessary

## Questions to Ask

- What happened?
- Who was involved?
- Is it suspicious?
- What evidence supports the alert?
- What should be investigated next?

## Key Insight

An alert is the beginning of an investigation, not the conclusion.
