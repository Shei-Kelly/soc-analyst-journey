What is a SOC Analyst?
A Security Operations Center (SOC) Analyst is responsible for monitoring, investigating, and responding to security alerts within an organization.
Typical Responsibilities
•	Monitor security alerts
•	Investigate suspicious activities
•	Analyze logs and events
•	Determine whether alerts are malicious or benign
•	Document findings
•	Escalate incidents when necessary
Tier 1 SOC Analyst
A Tier 1 SOC Analyst is the first line of defense and focuses on:
•	Alert monitoring
•	Alert triage
•	Initial investigations
•	Documentation
•	Escalation of confirmed threats
________________________________________
Event vs Alert
Event
An event is any activity recorded on a system.
Examples:
•	User login
•	File creation
•	Process execution
•	Account creation
Alert
An alert is generated when a security tool detects activity that may indicate malicious behavior.
Examples:
•	Multiple failed login attempts
•	Malware detection
•	Suspicious PowerShell execution
Key Difference
Event	Alert
Any recorded activity	Potential security concern
Generated by systems	Generated by security tools
Usually normal	Requires investigation

What is a SIEM?
SIEM stands for Security Information and Event Management.
A SIEM collects, stores, and analyzes logs from multiple systems and generates alerts when suspicious activity is detected.
Functions of a SIEM
•	Centralized log collection
•	Event correlation
•	Threat detection
•	Alert generation
•	Security monitoring
Examples
•	Wazuh
•	Splunk
•	Microsoft Sentinel

Windows Event Logs
Windows Event Logs provide records of activities occurring on a Windows system.
________________________________________
Event ID 4624 - Successful Login
Description
A user successfully authenticated to the system.
Why It Matters
Attackers who obtain valid credentials can generate successful login events.
Suspicious Indicators
•	Login at unusual hours
•	Login from unknown locations
•	Unexpected administrator logins
SOC Question
"Who logged in and was it expected?"
________________________________________
Event ID 4625 - Failed Login
Description
A login attempt failed.
Why It Matters
Can indicate brute-force attacks, password spraying, or credential stuffing.
Suspicious Indicators
•	Large number of failed logins
•	Repeated attempts from a single IP
•	Multiple accounts targeted
SOC Question
"Is someone attempting to gain unauthorized access?"

MITRE ATT&CK Framework
MITRE ATT&CK is a framework used to describe attacker behavior.
It helps SOC analysts:
•	Understand attacks
•	Classify incidents
•	Build detections
•	Improve investigations
________________________________________
Tactics vs Techniques
Tactic
The attacker's goal (WHY).
Examples:
•	Initial Access
•	Credential Access
•	Persistence
•	Privilege Escalation
Technique
The method used to achieve the goal (HOW).
Example:
•	Tactic: Credential Access
•	Technique: Brute Force
________________________________________
Important MITRE Tactics Learned
Initial Access
How attackers enter a system.
Examples:
•	Phishing
•	Exploiting vulnerabilities
•	Stolen credentials
________________________________________
Credential Access
How attackers obtain credentials.
Examples:
•	Brute force attacks
•	Password spraying
•	Credential dumping
________________________________________
Privilege Escalation
How attackers gain elevated permissions.
Examples:
•	Exploiting vulnerabilities
•	Using stolen administrator credentials
________________________________________
Persistence
How attackers maintain access.
Examples:
•	Creating user accounts
•	Installing backdoors
•	Scheduled tasks
