# Day 3 – Complete SOC Learning Summary

## Overview

Day 3 focused on developing core SOC analyst thinking skills, including alert analysis, log correlation, attack storytelling, MITRE ATT&CK mapping, and full incident investigation.

---

## 1. Alerts and Triage

Security alerts are notifications generated by security tools when suspicious activity is detected.

### Key Concepts:
- **Alert ≠ Attack**
- Alerts must be validated through investigation
- SOC analysts perform **triage** to determine severity and priority

### Important Terms:
- True Positive → Real malicious activity
- False Positive → Benign activity incorrectly flagged
- Severity Levels → Low, Medium, High, Critical

---

## 2. Log Correlation

Log correlation is the process of combining multiple logs to understand the full context of an event or attack.

### Example Chain:
- 4625 → Failed login
- 4624 → Successful login
- 4688 → Process execution
- DNS → External communication

### Key Insight:
Single logs are not enough. Context is required to understand attacks.

---

## 3. Attack Story Building

Attack storytelling converts logs into a human-readable incident narrative.

### Example:
Instead of listing logs, SOC analysts describe:
- What happened
- How it happened
- What the attacker did

### Benefit:
Helps non-technical stakeholders understand incidents clearly.

---

## 4. MITRE ATT&CK Mapping

The MITRE ATT&CK framework maps attacker behavior to standardized tactics and techniques.

### Key Mappings:
- 4625 → Credential Access
- 4624 → Initial Access
- 4688 → Execution
- 4720 → Persistence
- DNS/HTTPS → Command and Control

### Key Insight:
MITRE helps standardize how attacks are described.

---

## 5. Full Incident Investigation

A full SOC investigation includes:
- Alert analysis
- Log correlation
- Timeline building
- MITRE mapping
- Risk assessment
- Incident narrative
- Escalation decision

### Example Attack Chain:
4625 → 4625 → 4624 → 4688 → DNS → HTTPS

This may indicate:
- Brute force attempt
- Successful compromise
- Malicious execution
- External communication

---

## 6. Lessons Learned

- Alerts must always be investigated, not trusted blindly
- Logs must be correlated to understand full context
- Attackers leave multi-step traces, not single events
- MITRE ATT&CK improves communication and analysis
- SOC analysts must think in timelines, not isolated logs
- Clear communication is as important as technical analysis
